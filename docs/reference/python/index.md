# Python API Reference

These pages combine two layers:

- generated Markdown inventories that stay readable in the GitHub source view
- `mkdocstrings` output in the built docs site for the full symbol-level API reference

The `mkdocstrings` layer is generated from mirrored source modules in [docs_api](/C:/Users/el_de/Documents/New%20project/CMXsafeMAC-IPv6/docs_api).

Why the mirrored modules exist:

- the repo has several top-level files named `app.py`
- the source lives in component directories that are ideal for the runtime layout, but not ideal as import names for a docs tool
- the mirrored modules give the docs site stable module names without changing the production source layout

The mirrors are generated by:

- [sync-docs-api.ps1](../../../tools/sync-docs-api.ps1)
